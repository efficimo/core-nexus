BRIEFING OPÉRATIONNEL

Le Terminal de Combat n'est pas un monolithe. Ses composants fondamentaux —
système réactif, couche de stockage, protocole réseau, interface de
visualisation — ont vocation à vivre en dehors du Terminal, dans la
communauté open source.

Chaque librairie réside dans son propre dépôt GitHub indépendant.
Pas de monorepo. Pas de Turborepo. Pas de Changesets.
Chaque package évolue, se versionne et se publie de façon autonome.

STRUCTURE TYPE D'UN DÉPÔT

  Deux variantes selon que le package expose ou non des bindings React.

  ── VARIANTE A : package pur (zéro dépendance framework) ──────────────

  efficimo/<package>/
  ├── src/
  │   ├── index.ts
  │   └── *.ts
  ├── test/
  │   └── *.test.ts
  ├── scripts/
  │   └── postbuild.mjs
  ├── .github/
  │   └── workflows/
  │       ├── ci.yml          ← typecheck + tests sur push/PR main
  │       └── publish.yml     ← publish npm sur tag v*
  ├── package.json            ← name: "@efficimo/<package>"
  ├── tsconfig.json           ← base (types: [])
  ├── tsconfig.esm.json       ← build ESM → dist/esm/
  ├── tsconfig.cjs.json       ← build CJS → dist/cjs/
  ├── tsconfig.test.json      ← tests CJS → .test-out/
  ├── .gitignore
  └── README.md

  Packages concernés : deferred-promise, observable, storage, cipher,
                       workers, memento, monitor, plexus

  ── VARIANTE B : package core + bindings React ────────────────────────

  efficimo/<package>/
  ├── core/                   ← @efficimo/<package>
  │   ├── src/
  │   │   ├── index.ts
  │   │   └── *.ts
  │   ├── test/
  │   │   └── *.test.ts
  │   ├── scripts/
  │   │   └── postbuild.mjs
  │   ├── package.json        ← name: "@efficimo/<package>"
  │   ├── tsconfig.json
  │   ├── tsconfig.esm.json
  │   ├── tsconfig.cjs.json
  │   └── tsconfig.test.json
  ├── react/                  ← @efficimo/<package>-react
  │   ├── src/
  │   │   ├── index.ts
  │   │   └── use*.ts         ← hooks React
  │   ├── test/
  │   │   └── *.test.ts
  │   ├── scripts/
  │   │   └── postbuild.mjs
  │   ├── package.json        ← name: "@efficimo/<package>-react"
  │   │                          peerDeps: react, @efficimo/<package>
  │   ├── tsconfig.json
  │   ├── tsconfig.esm.json
  │   ├── tsconfig.cjs.json
  │   └── tsconfig.test.json
  ├── .github/
  │   └── workflows/
  │       ├── ci.yml          ← typecheck + tests core + react sur push/PR main
  │       ├── publish-core.yml   ← publish sur tag v*
  │       └── publish-react.yml  ← publish sur tag react-v*
  ├── .gitignore
  └── README.md               ← documente les deux packages

  Packages concernés : dock, plexus-ui

CONVENTIONS ÉTABLIES (issues de deferred-promise)

  BUILD
  - Dual ESM + CJS via deux tsc + postbuild.mjs (aucune dépendance externe)
  - dist/esm/package.json → { "type": "module" }
  - dist/cjs/package.json → { "type": "commonjs" }
  - Zéro bundler tiers (pas de tsup, rollup, vite lib mode)

  TESTS
  - node:test + node:assert/strict (natif Node 18+)
  - tsconfig.test.json compile vers .test-out/ en CJS
  - Fake timers via t.mock.timers pour les tests temporels
  - devDependencies : typescript + @types/node + undici-types

  CI/CD
  - actions/checkout@v6 + actions/setup-node@v6
  - Node 24
  - Trusted publishing OIDC npm (pas de NPM_TOKEN stocké)
  - Tag v* → publish / push main ou PR → CI

  PACKAGE.JSON
  - publishConfig.registry + publishConfig.access
  - prepublishOnly : build + typecheck
  - engines.node >= 18

  QUALITÉ
  - Symbol.species → return Promise sur toute extension de Promise
  - Pas de dépendances runtime

DÉCISIONS ARCHITECTURALES

  ✦ MULTI-REPO plutôt que MONOREPO
    Chaque librairie vit dans son propre dépôt git indépendant.
    Raison : autonomie complète de chaque package, pas de couplage
    outillage (Turborepo, Changesets, pnpm workspaces inutiles).
    Contrepartie : les dépendances inter-packages passent par npm —
    publier avant de développer les dépendants.

  ✦ BUILD NATIF tsc uniquement
    Aucun bundler tiers. Dual ESM/CJS via deux tsconfig + postbuild.mjs.
    Validé sur deferred-promise.

  ✦ TESTS NATIFS node:test
    Aucune dépendance de test externe (pas de Vitest, Jest, etc.).
    Validé sur deferred-promise (31/31 tests).

  ✦ TRUSTED PUBLISHING OIDC
    Pas de NPM_TOKEN stocké en secret GitHub.
    Flux : tag git → GitHub Actions → OIDC → npm.
